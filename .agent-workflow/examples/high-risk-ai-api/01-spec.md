# 后端 AI API 代理 SPEC

## 背景

前端需要调用 AI 服务生成简短摘要，但 API Key 不能暴露在浏览器端。系统需要通过后端代理调用 AI API，并对输入、超时、错误和返回结构做校验。

## 目标

新增后端接口 `POST /api/ai/summary`：

- 接收用户文本。
- 从环境变量读取 AI API Key。
- 调用 AI 服务生成摘要。
- 返回结构化 JSON。
- 不在前端或日志中暴露 API Key。

## 非目标

- 不实现用户登录。
- 不保存用户输入。
- 不做计费。
- 不做多模型切换。
- 不做复杂 prompt 管理平台。

## 调用路径

```text
Frontend
  -> POST /api/ai/summary
    -> Backend validates input
      -> Backend reads AI_API_KEY from environment
        -> Backend calls AI provider
          -> Backend validates response
            -> Frontend receives { "summary": "..." }
```

## 功能要求

- 请求体字段：`text`。
- `text` 必须是非空字符串。
- `text` 最大长度 4000 字符。
- API Key 只能来自后端环境变量 `AI_API_KEY`。
- AI 调用必须设置超时。
- AI 响应必须校验结构。
- 失败时返回可观测错误，不返回假成功。

## 边界情况

- 空文本：返回 400。
- 超长文本：返回 400。
- 缺少 `AI_API_KEY`：返回 500，并提示服务配置错误。
- AI 服务超时：返回 504。
- AI 服务返回非 JSON 或结构不符合预期：返回 502。
- AI 服务拒绝请求：返回 502，不泄露密钥。

## 技术选择

- 语言/框架：按项目现有后端技术栈。
- 测试：使用项目现有 API 测试框架。
- 包管理：按项目现有包管理器。
- 数据库：不涉及。
- 部署：涉及环境变量。

## 风险等级

L3

## 触发原因

- 涉及 AI Service。
- 涉及环境变量和 API Key。
- 涉及外部 API。
- 涉及用户输入和错误处理。

## 必须启用角色

- ①测试工程师
- ②开发工程师
- ③验收工程师
- ④质量工程师
- ⑤安全工程师
- ⑩风险审查官
- ⑥性能工程师
- ⑦文档工程师
- ⑧集成工程师
- ⑨部署工程师

## 验收标准

- 有失败测试覆盖正常路径、空输入、超长输入、缺少 API Key、AI 超时、AI 返回结构错误。
- 后端实现通过测试。
- 安全审查确认 API Key 不在前端、日志、响应中暴露。
- 风险审查确认外部服务失败路径可接受。
- 真实验证记录说明如何用测试环境 API Key 验证。
- 文档说明 `AI_API_KEY` 配置和接口调用方式。
