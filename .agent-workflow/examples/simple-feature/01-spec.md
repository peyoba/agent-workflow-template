# CLI --version 参数 SPEC

## 背景

当前 CLI 没有统一的版本输出方式，用户无法快速确认正在使用的工具版本。

## 目标

用户运行 `tool --version` 时，CLI 输出当前版本号并以状态码 0 退出。

## 非目标

- 不修改发布流程。
- 不新增配置系统。
- 不改变其他 CLI 参数行为。

## 用户路径 / 调用路径

```bash
tool --version
```

预期输出：

```text
tool 1.0.0
```

## 功能要求

- 支持 `--version` 参数。
- 输出格式为 `<工具名> <版本号>`。
- 成功时退出状态码为 0。
- 不影响已有参数。

## 边界情况

- 同时传入未知参数时，保持现有未知参数处理逻辑。
- 版本号来源如果项目已有版本常量，则使用已有常量。
- 如果项目没有版本常量，先使用最小实现，后续再接入包版本。

## 技术选择

- 语言/框架：按项目现有 CLI 技术栈。
- 测试：使用项目现有 CLI 测试方式。
- 包管理：按项目现有包管理器。
- 数据库：不涉及。
- 部署：不涉及。

## 风险等级

L2

## 触发原因

新增用户可见 CLI 行为，需要测试、开发、验收、质量检查。

## 验收标准

- 运行 `tool --version` 输出 `tool 1.0.0`。
- 命令退出状态码为 0。
- 相关 CLI 测试通过。
